## Agreed stack decisions (2026-06-11, maintainer-confirmed)

These are settled — executors and future advisors do not re-litigate:

1. **Turborepo** (not plain bun workspaces) — for Vercel zero-config deploys
   and Remote Caching.
2. **Scaffold-first**: structure comes from `bunx create-turbo@latest` and
   `bunx create-next-app@latest`; code is ported INTO scaffolds. Never
   hand-author package.json manifests; dependencies via `bun add` only.
3. **Latest versions always**: `bun add` without pins; major-version breaks
   are reported, not silently pinned (STOP conditions in plan 003).
4. **Type safety via zod**: `packages/schemas` is the canonical API contract;
   TS types are `z.infer` only. Plain shared interfaces rejected.
5. **Naming**: `apps/web` (portfolio backend + public site), `apps/desktop`
   (Tauri client). Packages namespaced `@repo/*`.
6. **Scope**: web + desktop only. `bookmark-extension/`,
   `deniz-nutrition-api/`, `macros/` stay out (gitignored).
7. **CI**: GitHub Actions + Dependabot, with a workflow that auto-repairs
   `bun.lock` on Dependabot PRs.
8. Trade-off accepted: fresh root git history; originals preserved with full
   history under `_archive/`.

## Findings considered and rejected

So nobody re-audits these:

- **SEC-01 "secrets committed in .env"**: FALSE — `.env` is gitignored and not
  tracked (`git ls-files` shows only `.env.example`); auditor read the local
  file. Local plaintext `.env` is standard practice. The tracked
  `.envoy/config.toml` was checked: 11 lines, no secrets.
- **BUG-01 "rate-limit check-then-increment race"**: the `findOneAndUpdate`
  uses an atomic aggregation-pipeline update; the over-limit `$pop` removes
  the just-appended timestamp. Design is sound.
- **SEC-11 "API key brute force"**: keys stored as SHA-256 hashes, compared
  by hash lookup; search space makes online brute force a non-issue.
- **SEC-08 "missing CSRF"**: better-auth ships CSRF protections and modern
  SameSite cookie defaults; admin API additionally accepts Bearer keys (CSRF
  proof). Downgraded to "verify better-auth cookie config once".
- **SEC-07 IDOR on blogs**: single-admin system by design.
- **SEC-10 "stack traces leak"**: `console.error` goes to server logs, not
  responses. Misread.
- **SEC-13 blog slug uniqueness**: LOW confidence, functional not security.
- **SEC-03 IMAP password input validation**: library handles it; negligible upside.
- **jobs/* route auth**: checked — all five cron routes verify a Bearer token
  against env. Clean.
- **Plain shared-interface package** (first draft of plan 004): superseded by
  the zod-schema decision; draft deleted.
- **History-preserving `git subtree` import** (first draft of plan 003):
  superseded by scaffold-first decision; history lives in `_archive/` instead.
- **BUG-03 "kanban optimistic update not rolled back"**: RE-VERIFIED @ `bbaedfe`
  (`apps/desktop/app/dashboard/kanban/_components/kanban-board.tsx`). On a failed
  reorder/move PATCH, `handleDrop` calls `fetchBoard()` which refetches
  authoritative state from the server — the optimistic UI is reconciled, not
  left stale. Refetch-on-failure is an acceptable rollback strategy. Not a bug.
- **BUG-06 "kanban window listeners leak on mid-drag unmount"**: RE-VERIFIED @
  `bbaedfe` (same file). The global `mousemove` tracker effect (`[]` deps) and
  the per-drag `mousemove`/`mouseup` effect (`[dragging]` deps) both return
  cleanup functions that `removeEventListener`; React runs them on unmount.
  No leak. Not a bug.
